infdev3 : crashtest

Various objects have been implemented.
The player can move but its movements aren't constrained to the desired behavior yet.
Beat dependant object are implemented. This mean, if we can create a level and attach it a song and BPM (Beats Per Minute), any beat dependant object will work in sync with the song.
Cannons object are fully implemented. They are configurable and depends on the current songs BPM.
Jumpers object have been partially implemented. They do make the player jump but they aren't very configurable yet. They are beat dependant and their timing are configurable.

Early network implementation. Able to connect 1 (or theorically more) client to host. Connection is established and works. However it breaks some scripts and no interaction is supported.
